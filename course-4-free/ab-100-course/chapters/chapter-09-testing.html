Prompt metrics & patterns

Five quality metrics for evaluating prompts. Plus enterprise prompt patterns that consistently produce better output.

PROMPT QUALITY
5 quality metrics
  • Accuracy — Output correctness, intent alignment.
  • Consistency — Reliable quality across repeated runs.
  • Relevance — Focus on intended task only.
  • Format compliance — Output follows required structure.
  • Tone alignment — Matches organization or audience standards.
ENTERPRISE PATTERNS
Production prompt patterns
  • Order matters: Goal → Context → Constraints.
  • Use action verbs — summarize, analyze, rewrite, evaluate.
  • Structured instructions — lists, steps, format spec.
  • Avoid multi-task prompts — one job per prompt.
  • Add exclusions — "what NOT to include" if needed.

Scenario discussion

Design the test plan. Healthcare context — clinical accuracy is non-negotiable.

Scenario · Pair Discussion

Acme Healthcare's Discharge Summary Agent

"Acme Healthcare runs an 80-bed hospital across 12 departments. They've built a Copilot Studio agent that generates patient discharge summaries from EHR notes — saving nurses hours per discharge. Goes to production in 6 weeks. Subject to HIPAA + state regulations. Clinical accuracy is critical (wrong dosage = patient harm). The agent uses a custom medical-terminology model. About 200+ nurses will use prompts they craft themselves. The flow spans EHR → Agent → Pharmacy system → Billing. The team wants to use Copilot to help generate test cases."
Discuss with your partner
  • Which of the 4 testing types matters most here? Why? What can't you skip?
  • The custom medical model — what validation thresholds would you set? Same as Microsoft's example, or stricter? Why?
  • 200+ nurses crafting their own prompts — how do you validate that prompts produce safe output without slowing down adoption?
  • EHR → Agent → Pharmacy → Billing. Design the E2E test scenario — what integration points, exception paths, AI decision validations?
  • How would you use Copilot to scale test case creation responsibly in a clinical context?
Guide · suggested points
All 4 testing types matter — Safety & Compliance is non-negotiable
Safety & Compliance: Required by HIPAA. Sensitive data protection, role-based access, DLP, rejection of disallowed instructions — zero tolerance for violations. Scenario-based: Test ambiguous EHR entries, multi-condition patients, drug interactions. Real clinical scenarios from de-identified historical data. Performance: Hospital workflows are time-pressured — latency under 3 seconds matters. Usability: Nurses can't refine prompts during a discharge — first-shot accuracy is critical.
Stricter thresholds — life-critical context
Accuracy: ≥99% (vs Microsoft's ≥90% example) — clinical decisions can't tolerate 10% error. Hallucination rate: Effectively 0% — any fabricated dosage, allergy, or instruction is a patient safety incident. Sensitive output blocking: 100% — non-negotiable for HIPAA. Latency: <3 seconds acceptable — clinical workflow tolerates this. The general principle: Higher stakes → tighter thresholds. Test thresholds reflect business risk, not template defaults.
Validate prompts without slowing adoption
Don't: Force every nurse to validate every prompt — they'll work around it. Do: Build a validated prompt library that 80% of cases use. Prompts validated by clinical SMEs in advance. For new prompts: A/B testing in a sandbox before broad use. Scenario-based validation across patient types. Automated guardrails: The agent should reject unsafe prompts regardless of who wrote them — guardrails, not prompts, enforce safety.
E2E test design for EHR → Agent → Pharmacy → Billing
Trigger: Patient admitted, EHR records created. Discharge order placed. Integration points to test: EHR data sync, agent grounding accuracy, pharmacy order generation, billing code propagation. AI decision validations: Discharge summary accuracy, dosage extraction correctness, allergy flag preservation. Exception paths: Conflicting medications, missing data, after-hours discharge, multi-physician overrides. Negative tests: Try to make the agent leak PHI to billing. Try to bypass dosage limits. Verify guardrails hold.
Copilot for test generation in a clinical context
Use Copilot for: Generating test case templates, drafting edge-case scenarios, creating de-identified test data variations. Don't use Copilot for: Final test approval. All clinical scenarios must be reviewed by a qualified medical SME. Prompt strategy: Provide HIPAA rules, dosage guidelines, formulary in context. Set quality expectation: "Include at least 3 negative tests per scenario." Maturity goal: Level 3 — versioned prompt library reviewed quarterly by medical and compliance teams. Don't push to L4 (CI/CD integrated) until L3 is stable.

Chapter 9 — key takeaways

Six things to carry forward into Chapter 10, where we cover ALM (Application Lifecycle Management) for AI solutions.

1
AI testing is fundamentally different — probabilistic outputs require new practices. Test variance, not just correctness.
2
Agent testing — 4 types mandatory: Scenario-based · Performance · Safety & Compliance · Usability.
3
Model validation — 5 dimensions + threshold matrix. Customize thresholds by domain risk.
4
Prompts are system components — 4 components · 8-step lifecycle · 5 quality metrics. Treat them as code.
5
E2E multi-app — map cross-app data flow, not module-by-module. Each integration point = failure mode.
6
Copilot for tests — Goal + Context + Constraints + Quality. Climb the maturity ladder · target L4 with CI/CD.
