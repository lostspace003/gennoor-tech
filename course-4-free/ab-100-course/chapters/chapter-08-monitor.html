Monitoring operating model

Four practices for production-grade monitoring. Skip any of them and monitoring becomes whatever the on-call engineer remembers to check.

1
Establish Operating Model

Roles, ownership, accountability for AI operations.

What to define

  • Standardized metric definitions
  • Baselines & trends

  • Latency thresholds

Human-in-the-loop quality checks beyond telemetry.

Continuous Improvement

Insights translated into measurable improvements.

What to drive
  • Failure pattern analysis

Four tuning strategies

Each tuning strategy targets a specific layer. Match strategy to root cause — tuning prompts when the issue is integration just wastes time.

📚
Knowledge Tuning
Strategy 1 · most common
  • Add or update content to fill gaps
  • Remove outdated information
  • Refine knowledge structure for retrieval
  • Re-index when corpus changes significantly
⚛
Behavioral Tuning
Strategy 2 · prompt engineering
  • Adjust orchestration steps
  • Add clarifying instructions
  • Introduce fallback strategies
  • Refine guardrail rules
⚡
Performance Tuning
Strategy 3 · perceived speed
  • Optimize connectors and external calls
  • Reduce unnecessary workflow steps
  • Address slow processing logic
  • Trim oversized payloads
🛡
Governance Tuning
Strategy 4 · compliance-aware
  • Review DLP, sensitivity, access rules
  • Align with compliance requirements
  • Preserve logging and audit trail
  • Document policy changes

Three metric categories + drift

Operational metrics measure infrastructure. Quality metrics measure reasoning. User-centered metrics measure success. Plus drift indicators specific to generative AI.

CATEGORY 01 · INFRASTRUCTURE

Operational

LatencyTime to process agent requests.
ThroughputVolume of completed runs per period.
Error rateProportion of failed/incomplete tasks.
Resource useCompute, memory, tokens consumed.
CATEGORY 02 · REASONING

Quality & Reasoning

Response accuracyAlignment with expected outputs.
Knowledge coverageSurfaces correct domain content.
Action effectivenessMulti-step tasks complete correctly.
CATEGORY 03 · USER OUTCOMES

User-Centered

SatisfactionUser feedback trends and sentiment.
AbandonmentDrop-off during agent workflows.
Task completion ★Users achieve intended outcomes — gold standard.
⚠ Model drift indicators · specific to generative AI
Pattern shifts: Response patterns drift from established norms.
Accuracy decline: Recurring tasks start producing weaker results.
Hallucination uptick: Off-topic or fabricated responses increase.

Scenario discussion

Diagnose, prioritize, and tune. Telemetry says one thing. Users say another.

Scenario · Pair Discussion

MegaRetail Sales Helper — healthy metrics, dropping adoption

"MegaRetail launched their Copilot for Sales agent 3 months ago for 1,200 sales reps. Telemetry looks healthy — see the dashboard. But weekly active users have dropped from 60% to 35%. User satisfaction surveys show frustration. Conversation transcripts show many users abandoning at the proposal-drafting step. Some reps complain the agent gives 'wrong' pricing recommendations. A few governance alerts about sensitive customer data appearing in summaries."

Discuss with your partner
  • Where do you start the diagnosis — telemetry, backlog, or transcripts? Why?
  • What does the gap between healthy operational metrics and dropping adoption tell you about the root cause?
  • Categorize the issues into backlog domains. Which tuning strategies would you apply, and in what order?
  • What monitoring would you put in place to catch this earlier next time?
Guide · suggested points
Start with transcripts (not telemetry)
Why transcripts: Healthy telemetry plus dropping adoption is the canonical "telemetry blind spot" pattern. The infrastructure is fine — the problem is user-facing. Transcripts reveal what telemetry can't: Misunderstood intents, abandoned flows, unmet expectations. Telemetry confirms scale: Once transcripts surface the issue, telemetry tells you how widespread.
What the metrics gap tells you
Operational health is fine — agent is up, fast, succeeding from the system's perspective. User-centered metrics are failing — abandonment, satisfaction, weekly active. The agent technically completes tasks but doesn't deliver value. Diagnosis: The "success rate" metric is measuring the wrong thing. It's counting technical completions, not user-perceived success.
Backlog categorization + tuning order
Knowledge: "Wrong pricing recommendations" → Knowledge tuning. Refresh pricing data sources. Priority 1 — high impact, low effort. UX: "Abandoning at proposal-drafting step" → Behavioral tuning. Clarify the flow, add fallbacks. Priority 1. Governance: "Sensitive data in summaries" → Governance tuning. Tighten DLP, sensitivity label enforcement. Priority 1 — compliance risk. Performance: Skip. Latency is fine. Don't waste cycles tuning what isn't broken.
Better monitoring for next time
Add user-centered metrics to dashboards: Task completion rate (gold standard), abandonment rate by workflow step. Set up monthly transcript reviews: Sample-based human-in-the-loop quality checks. Alert on adoption drift: Threshold alert when weekly actives drop more than 10% week-over-week. Drift indicators: Watch for response pattern shifts on recurring tasks like pricing recommendations.
