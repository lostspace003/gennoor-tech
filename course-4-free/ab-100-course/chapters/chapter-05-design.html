The six tenets of Responsible AI

Microsoft's published Responsible AI principles. Every agent on the Microsoft stack must satisfy all six. Skip any one and you create real organizational risk.

TENET 01
⚖

Fairness

Treat all people fairly. Avoid harmful biases. Ensure equitable outcomes across demographics.

TENET 02
⛨

Reliability & Safety

Operate reliably under expected and unexpected conditions. Continuous monitoring. Validate regularly.

TENET 03
🔒

Privacy & Security

Protect personal data, uphold privacy, follow strict security protocols, meet compliance.

TENET 04
⊕

Inclusiveness

Empower everyone. Avoid barriers. Accessibility and inclusive design principles built in.

TENET 05
◐

Transparency

Users understand how AI works. Explain data reliance and decision processes. Build trust through clarity.

TENET 06
⌖

Accountability

People — not machines — are responsible for AI outcomes. Guardrails, governance, human oversight.

Three agent types — when to use each

All three live in Copilot Studio. Choose based on how the work happens — driven by a user, by an event, or by conversation.

Type 1

Task Agents

Complete specific tasks on user demand
Designed for
  • Automate order entry
  • Summarize customer inquiries
  • Retrieve inventory status
  • Create & assign work items
Use whenUser-driven. Specific outcome. Repeatable workflow.
Type 2

Autonomous Agents

Work independently, event-driven
Designed for
  • Onboarding workflows
  • Scheduled report summaries
  • Customer service triage
  • Multi-step automation chains
Use whenEvent-driven. Multi-step. Runs without user.
Type 3

Prompt-Response Agents

Conversational, NLU + topics + generative
Designed for
  • FAQ & support agents
  • Guided assistance
  • Knowledge Q&A
  • Multi-turn dialog with branching
Use whenDialog-driven. Knowledge retrieval. Branching logic.

Three paths — NLU vs CLU vs Generative AI

Three language understanding modes in Copilot Studio. Pick the simplest one that meets the requirement. Don't reach for generative when NLU will do.

▪ LEAST COMPLEX · CHEAPEST

Standard NLU

Intent and entity models. Rules, patterns, structured training datasets. Deterministic.

Use when
  • Fixed phrases ("reset password")
  • Regulated processes need precision
  • Structured user queries
  • Deterministic responses required
ExampleCall center bot for password reset, balance check, ticket routing.
▪▪ MODERATE · TRAINABLE

Azure CLU

AI-driven classification. Trainable. Better entity extraction. Handles linguistic variability within bounds.

Use when
  • Variable phrasing, defined topics
  • Need scalable trainable models
  • Multilingual scenarios
  • Continuous accuracy improvement
ExampleField service bot understanding "device broken" / "won't start" / "frozen".
▪▪▪ MOST POWERFUL · MOST EXPENSIVE

Generative AI Orchestration

LLMs dynamically interpret prompts, retrieve knowledge, call actions. Open-ended dialog.

Use when
  • Unstructured user input
  • Summarization, rewriting, content gen
  • Multi-turn reasoning
  • Enterprise grounding required
ExampleSales assistant summarizing CRM, drafting emails, answering long-form Qs.

Four core mechanisms in Copilot Studio

When designing prompt-and-response agents, you have four building blocks. Combine them — most real agents use all four.

⚡

Generative Answers (NLU Boost)

When a topic doesn't match a user query, generative answers retrieve from knowledge sources and generate a response.

Use whenQuestion not covered by topics; need answers from knowledge sources.
⊟

System Topics

Built-in topics — greeting, fallback, escalation, disambiguation, error handling. Consistent UX scaffolding.

Use whenStandard scenarios — greetings, errors, escalations, ambiguity.
◇

Condition Nodes

Branch dialog using comparisons, variables, AND/OR logic, Power Fx formulas. Multiple branches.

Use whenVIP vs standard customer · form input · conversation context branching.
◉

Event Triggers

Act without user prompts — file uploads, task completion, Dataverse changes, schedules.

Use whenReactive behavior · automation · scheduled tasks. Watch billing impact.

Agent flows — automation inside Copilot Studio

Agent flows run natively inside Copilot Studio with no Power Automate license required. Same trigger-and-action pattern, optimized for agent execution.

What are agent flows?

Native automation inside Copilot Studio. Two primitives — triggers and actions. No Power Automate license required.

  • End-to-end process visibility
  • Uses Copilot Studio capacity
  • Natural-language authoring
  • AI-driven conversational automation
Trigger
How the flow starts

Events that initiate flow execution.

  • Manual user request
  • Scheduled / recurrence
  • System event (Dataverse row, file upload)
  • Another agent calling
Action
What the flow does

Operations performed once triggered.

  • Send email · post message
  • Update / create Dataverse records
  • Call connectors and APIs
  • Trigger downstream workflows

D365 Copilot — four customization layers

Customizing Copilot in Dynamics 365 isn't one thing — it's four. Apply all four layers to make Copilot reflect your customer's specific operations, vocabulary, and compliance.

01
Business Terms & Domain Language

Standardized organizational vocabulary stored in Dataverse fields, option sets, metadata.

Example: Define "Tier-1 Account" vs "Strategic Partner" with explicit field values.
02
Prompt & Output Settings

Adjust tone, structure, and required fields for AI-generated content.

Example: Force compliance disclaimers on every generated insurance quote.
03
Data Scope & Field Configuration

Limit Copilot's access to relevant, up-to-date data fields and entities.

Example: Exclude archived contacts and 2019-vintage notes from case summaries.
04
Surface Integration

Display Copilot outputs in forms, dashboards, workflows. Custom connectors via OAuth 2.0 + Entra ID.

Example: Embed Copilot summary in case form header; use Entra OAuth for connector auth.

Scenario discussion

Design an end-to-end agent. Choose type, components, language mode, and customizations.

Scenario · Pair Discussion

FieldCore wants an HVAC technician support agent

"FieldCore has 600 HVAC technicians in the field. They use a mobile app and call a help desk when stuck. Most calls are about diagnosing 30 common error codes — but technicians describe each problem differently. Repair manuals live in SharePoint. Some manuals are confidential to commercial accounts. The CIO wants to ship in 10 weeks. Help desk wants to reduce calls 40%. Compliance wants every recommendation audited."
Discuss with your partner
  • Which agent type — Task, Autonomous, or Prompt-Response? Justify.
  • Which language understanding mode — NLU, CLU, or Generative AI? Why?
  • List the five components — Goals, Triggers, Knowledge, Actions, Safety Rules — for this specific agent.
  • Which Responsible AI tenets apply most strongly? What success KPIs would you track?
Guide · suggested points
Agent type — Prompt-Response
Prompt-Response agent — technician asks a question, agent diagnoses. Conversational, knowledge-grounded. Not autonomous — the technician initiates each interaction. Not pure task — uses dialog, branching by symptoms, retrieval from manuals.
Language understanding — Azure CLU + Generative
Azure CLU for the 30 known error codes — technicians describe them dozens of ways. CLU classifies into the right code. Generative orchestration for the long-tail repair guidance — pull from SharePoint manuals, summarize into actionable steps. Standard NLU insufficient — too much phrasing variability for fixed patterns.
The five components
Goals: "Diagnose error code from symptoms" / "Provide repair steps from manuals" / "Escalate when uncertain." Triggers: Technician message via Teams mobile or Copilot Studio embedded widget. Knowledge: SharePoint repair manuals (scoped by account confidentiality), Dataverse error code library, technician profile (account access). Actions: Create case in Dataverse on escalation, log diagnosis, request senior tech via Teams handoff. Safety Rules: Never authorize part shipment over $X. Always cite manual section. Escalate any safety-related issue (refrigerant, electrical) to human.
RAI tenets & KPIs
Reliability & Safety — wrong diagnosis = damaged equipment or worse. Test extensively before launch. Privacy & Security — confidential manuals must respect account-level permissions. Transparency — every recommendation cites manual section. Accountability — technician makes final call; agent assists. KPIs: 40% call reduction (Operational Efficiency) · ≥85% recommendation acceptance (Quality) · <3s response (UX) · 100% audit trail (Risk & Compliance).
