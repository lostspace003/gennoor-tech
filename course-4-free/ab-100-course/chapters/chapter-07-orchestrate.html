Three D365 AI experience models

How Copilot shows up in Dynamics 365. Each model has a different interaction shape. Production deployments typically combine all three.

Model 1

Sidecar Copilot

Conversational chat panel beside the app
Common capabilities
  • Generative help & explanations
  • Workflow summaries
  • Natural language data queries
  • Action invocation via chat
Use whenUsers want to ask rather than navigate. Conversational over click-driven.
Model 2

Embedded Copilot

Contextual intelligence inside workspaces
Common capabilities
  • PO change analysis
  • Collections summaries
  • Demand planning insights
  • Supplier draft assistance
Use whenIntelligence needs to live in the form. In-flow assistance for power users.
Model 3

External Orchestration

Beyond the UI — cross-app automation
Common capabilities
  • Cross-application workflows
  • Role-specific Copilots in Teams
  • Workflow routing
  • Automated notifications
Use whenProcess spans D365 and other systems. Triggered events, not user actions.

Sales vs. Service — configuration workflows

Same architecture pattern, different data scope. Configuration is sequential. Field mapping is the make-or-break step in both.

Copilot for Sales

Configuration workflow + KPIs
Configuration steps
  1. Enable Copilot for Sales (M365 + D365)
  2. Connect CRM via approved connectors
  3. Map opportunity, account, activity fields
  4. Define least-privilege permissions
  5. Configure content sources (email, opportunity, proposal, meeting prep)
Sales KPIs Email/meeting prep time saved · lead-to-opportunity speed · proposal quality consistency.

Copilot for Service

Configuration workflow + KPIs
Configuration steps
  1. Connect case management engine (D365 CS or equivalent)
  2. Validate knowledge article repositories — structure, quality, labels
  3. Enable Copilot actions (summarize · lookup · guide)
  4. Integrate Power Automate (escalation · routing · approvals)
  5. Establish role-based access for sensitive actions
Service KPIs Case handle time · first-contact resolution · knowledge retrieval speed · article accuracy.

Scenario discussion

Choose the right mix of prebuilt agents and configurations. Defend your reasoning.

Scenario · Pair Discussion

Northwind Manufacturing wants AI across the enterprise

"Northwind has 4,000 employees. They run D365 Finance, D365 Supply Chain, D365 Customer Service, and Microsoft 365. Sales reps want help with proposals (in Outlook) and CRM updates. Customer service has 200 agents wanting case summarization and sentiment-aware routing. Finance team wants help with collections and statement summaries. Procurement wants vendor compliance lookups that combine F&O data with SharePoint contracts. Project deadline: 12 weeks. Budget is tight."
Discuss with your partner
  • Which prebuilt D365 capabilities apply directly to each business team? Which experience model — sidecar, embedded, external?
  • Sales reps need help in Outlook — which M365 Copilot configuration solves this? What are the data prerequisites?
  • Procurement vendor lookups span F&O + SharePoint. What's the right architecture — plugins, knowledge sources, or both?
  • For a 12-week timeline + tight budget, what's the right mix of prebuilt vs Copilot Studio extension vs custom Foundry?
Guide · suggested points
Prebuilt D365 capability mapping
Customer Service: Agent Hub + autonomous Service Agents (Customer Intent + Case Management) + Copilot in Contact Center for embedded summarization. Sidecar + embedded models. Finance: Collections coordinator summaries, customer page summaries, statement posting summaries. Embedded model — lives inside the Finance workspaces. Supply Chain: Supplier communication agent + AI-augmented demand planning. Embedded + external orchestration for cross-system supplier comms.
Sales reps in Outlook — M365 Copilot for Sales
Configuration: Enable Copilot for Sales connecting M365 + D365. Map opportunity, account, activity fields. Define seller-role least-privilege permissions. Data prerequisites: Validate CRM data is connected and synchronized. Sensitivity labels on sales documents in OneDrive/SharePoint. Standardized opportunity/account/contact fields. Deliverable: Email summarization, opportunity review, proposal drafting, meeting prep — all in Outlook.
Procurement vendor lookups — both knowledge sources AND plugins
Knowledge sources: SharePoint contract library connected as external knowledge source to F&O agent. Plugins: Required if procurement wants to update vendor terms — write-back operations always need a plugin. Combined response pattern: "Vendor X is approved in F&O (data) and their compliance certificate (SharePoint) expires in 30 days. Update payment terms?" → plugin action.
12-week / tight budget — heavy prebuilt, minimal Copilot Studio, no Foundry
Prebuilt first (~80% of value): All D365 prebuilt agents + M365 Copilot for Sales + M365 Copilot for Service. These ship configured — weeks of effort, not months. Copilot Studio extension (~15%): Custom topic for procurement combined-source query. Custom plugin for vendor term updates. Foundry custom (0%): Skip. None of the indicators hit strongly enough — domain isn't specialized, decisions aren't safety-critical, no sovereignty mandate. The architect's job: Defend prebuilt-first against the inevitable internal voice that says "let's just build it."

Chapter 7 — key takeaways

Six things to carry forward into Chapter 8, where we cover analyzing, monitoring, and tuning AI solutions.

1
D365 — 3 experience models: Sidecar (chat) · Embedded (in-form) · External (cross-app). Combine in production.
2
D365 Customer Service — Agent Hub + 4 autonomous agents + Copilot in Contact Center. Most mature surface.
3
M365 agents — 8-item readiness checklist. Business value · identity · data · actions · security · change · measurement · support.
4
M365 agent design — 5 steps: Frame job · Map I/K/A · Define guardrails · Prototype · Operationalize.
5
M365 Sales/Service — field mapping is critical. Sales: opportunity/account/activity. Service: validated knowledge repos.
6
Power Platform — AI Hub (govern) · Copilot (create) · AI Builder (predict/extract) · Copilot Studio (chat). Match to scenario.
