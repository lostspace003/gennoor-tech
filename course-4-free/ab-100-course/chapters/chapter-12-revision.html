Architect mental models

When you can't remember the exact framework, these rules of thumb will get you to the right answer.

RULE 01
SaaS-first, custom-last

If a prebuilt option fits the requirement, that's usually the answer. Custom Foundry models are last resort, not first instinct.

RULE 02
Least privilege wins

Any option that says "narrowest scope" or "managed identity" beats options that grant broad access. Always.

RULE 03
Layered controls beat single

Any option proposing one control to solve a security problem is wrong. The exam favors defense in depth.

RULE 04
Match rigor to risk

Don't apply healthcare-grade validation to a sales chatbot. Stricter for regulated, looser for low-risk. Both answers exist on the exam.

RULE 05
Audit metadata, not content

Logging prompt content is a leak vector. Logging metadata — who, when, model version, outcome — is auditable without exposure.

RULE 06
Bias toward disable

In incident response, when in doubt, disable first, investigate second. The exam rewards safety over operational continuity.

Common exam pitfalls

Patterns that catch unprepared candidates. Recognize the trap, save the wrong answer.

▸ Six traps to recognize
⚠ Over-privileged role assignments

"Owner at subscription scope" or "Contributor at resource group" answers are almost always wrong. Tight scope wins.

⚠ Reaching for custom too early

If a prebuilt agent or D365 Copilot pattern fits, that's the SaaS-first answer. Don't build what Microsoft already built.

⚠ Single-layer security answers

"Use only DLP" or "Use only MFA" — wrong. The right answer combines identity + data + observability + threat.

⚠ Performance over compliance

Don't pick "cross-region for latency" when the data is regulated. Residency wins. Always.

⚠ "Bigger model = better"

The model router is often correct over "always use the most capable model" — cost, latency, and routing matter.

⚠ "Fully autonomous" answers

Removing humans from sensitive workflows is dangerous. Approval workflows beat "let the agent decide" for regulated tasks.
