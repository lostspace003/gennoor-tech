Why ROI criteria matter

AI projects fail for predictable reasons. All four are about not having a business case — or having a wrong one. Get this right at the start and your project survives.

!
Overestimating productivity gains

Vendor demos show 50% savings. Reality often delivers 15%. Anchor on telemetry, not marketing.

!
Underestimating ops & maintenance

Model inference is cheap. Data prep, monitoring, retraining, governance is where the budget goes.

!
No clear measurement framework

Without a 'no-AI' baseline, you can't prove value after deployment. Establish baseline first.

!
Can't articulate value to leadership

Engineers love the tech. Executives want the dollar number. Translation is the architect's job.

Three strategic options — when to choose each

Build, buy, or extend. Each has a clear sweet spot. The wrong choice burns months and millions.

Option 1

Build

Create internally · maximum control
When to build
  • AI defines competitive advantage
  • Regulatory needs unmet by SaaS
  • Data sensitivity demands full control
  • Strong internal AI/ML capability
Option 2

Buy

Acquire SaaS · fastest path
When to buy
  • Efficiency matters more than custom
  • Standard processes meet your needs
  • Low internal AI maturity
  • Vendor ecosystem provides upgrades
Option 3

Extend

Customize Copilot · hybrid path
When to extend
  • Want Copilot adopting your logic
  • Domain knowledge added via grounding
  • Base model good but needs context
  • Default path for most enterprise needs

Why model routing matters

Modern AI apps use multiple models — general LLMs, small language models, fine-tuned models, task-specific models. The router decides which one handles each request.

$
Optimize Cost

SLMs for simple tasks. LLMs only when needed. 80/20 rule applies — 80% of requests don't need GPT-4-class reasoning.

⚡
Improve Performance

Low-latency to fast models. Domain-specific to fine-tuned. Right-size each request to its real needs.

⛨
Increase Reliability

Automatic failover. Version-aware routing. Production-grade resiliency without rewriting application code.

⇆
Hybrid AI Strategies

Mix foundation + fine-tuned + custom. The router becomes the integration layer for build/buy/both strategies.

20 / 23

Four types of routing rules

Use these in combination. A real production router runs static rules first, then weighted A/B, with fallback safety nets and version-based gradual rollouts.

RULE TYPE 01

Static Rules

Deterministic if-then routing. Predictable cost and behavior.

if task = classification
→ use SLM
if domain = finance
→ use fine-tuned model
RULE TYPE 02

Weighted Routing

Distribute traffic across models for A/B testing and gradual rollout.

v1 → 90% traffic
v2 → 10% traffic
shift gradually based on KPIs
RULE TYPE 03

Fallback Routing

If primary model fails or rate-limits, route to backup model automatically.

primary: gpt-4-turbo
fallback-1: gpt-4o-mini
fallback-2: phi-3-medium
RULE TYPE 04

Version-Based Routing

Route by model version — latest stable, experimental, or pinned.

production → stable
canary → experimental
compliance → pinned-v1

Scenario discussion

Real numbers, real decision. Calculate the ROI and recommend Build · Buy · Extend.

Scenario · Pair Discussion

InsureCo wants to triage claims with AI

"InsureCo processes 80,000 claims a year. Each claim takes a junior adjuster 22 minutes on average to read, categorize, and route. Loaded labor rate is $52 an hour. Leadership wants to deploy an AI agent to triage simple claims. Vendor pitched a custom build at $1.2M with 11-month timeline. Your platform team thinks Copilot Studio with custom topics + Azure AI Search grounding could ship in 8 weeks."

  • If the agent saves 14 minutes per triaged claim — what's the annual labor savings?
  • Build option: $1.2M, 11mo. Extend (Copilot Studio): est. $250K Year-1 TCO. Compute ROI %, payback for each.
  • Apply the weighted scoring matrix — does the math support Build or Extend?
  • What model routing rule would you put in place to keep ongoing costs down?
Guide · suggested points
Annual labor savings calculation
14 min ÷ 60 × 80,000 × 70% × $52 = ~$679,000 / year
That's labor only — adjuster time freed for higher-value work. Add error-cost-avoided + faster cycle time → realistic upside ~$850K–$900K.
ROI & payback — Build vs Extend
Build: Year-1 net = $679K − $1.2M = −$521K loss · ROI = −43% · payback ~21mo
Extend: Year-1 net = $679K − $250K = +$429K gain · ROI = 172% · payback ~4.4mo
Extend wins decisively on year-1 economics. Build catches up only in Year 3+ assuming the customization compounds.
Weighted scoring assessment
Strategic fit: claims triage is operational, not differentiating — Extend wins. Time-to-value: 8 weeks vs 11 months — Extend dominates. TCO: $250K vs $1.2M — Extend dominates. Customization: Copilot Studio + Azure AI Search handles claim ground truth — Extend sufficient. Recommendation: Extend. Push back firmly on the vendor's $1.2M pitch.
Model routing rule
Static rule: simple-category claim → SLM (Phi-3 or similar). Complex/edge case → GPT-4-class. Fallback: if primary rate-limits, route to backup SLM. Outcome: ~80% of claims hit the cheap path → token costs drop 60–70% vs single-LLM design.

Chapter 4 — key takeaways

Six things to carry forward.

1
Five ROI categories: Productivity · Cost Savings · Revenue · Risk Reduction · Strategic Value.
2
Five TCO drivers: Infrastructure · Development · Data Prep · Expertise · Operations. All five, every project.
3
Core formula: Annual Benefit = (min/run ÷ 60) × runs/year × labor rate + error/throughput effects.
4
Extend wins by default. Build only when AI is your competitive advantage. Buy only when speed beats fit.
5
Use the weighted scoring matrix: 5 criteria × 3 options. Sum columns. Defensible to leadership.
6
Model Router cuts 60–80% of token cost. SLM for simple, LLM for complex, fallback for resilience.
