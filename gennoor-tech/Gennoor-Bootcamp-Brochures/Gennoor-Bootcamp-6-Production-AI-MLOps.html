Production AI & MLOps

Course Overview

This 5-day bootcamp covers the complete MLOps lifecycle — from experiment tracking and model versioning through CI/CD pipelines, containerized deployment, and production monitoring across Azure ML, AWS SageMaker, and open-source tools. Participants design and implement a fully automated ML pipeline from data ingestion to live inference by the end of the program.

85% of ML models never make it to production. This bootcamp exists to change that number for your organization. You'll learn the engineering practices, tooling, and automation patterns that separate experimental notebooks from reliable, scalable, monitored AI systems. Designed for teams that already know how to build models — and now need to ship them.

🔧

🏗️

Teams building internal ML platforms and infrastructure

👨‍💻

Data scientists learning to productionize their own models

Learning Outcomes

1
Design end-to-end ML pipelines — data ingestion, feature engineering, training, evaluation, and deployment as automated workflows
2
Implement CI/CD for ML — automated testing, model validation, and deployment using GitHub Actions and Azure DevOps
3
Deploy models at scale — containerize with Docker, serve with FastAPI, deploy to Azure ML, AWS SageMaker, and Kubernetes
4
Monitor production models — detect data drift, model degradation, and performance anomalies with automated alerting
5
Version everything — track experiments with MLflow, version datasets with DVC, manage model registries across environments
6
Build across clouds — implement MLOps patterns on Azure ML, AWS SageMaker, and GCP Vertex AI with open-source tooling

Prerequisites

Python programming (intermediate), basic ML model building experience (Bootcamp 4 or equivalent), familiarity with Git and command line. Docker experience is helpful but not required — covered on Day 1.

Course Curriculum

5-Day Curriculum

Interactive format with live coding, hands-on labs, and project-based learning. Schedule and session order are fully customizable based on your organization's needs and time zones.

01
MLOps Foundations — Experiment Tracking & Versioning
Lecture + Live Coding + Hands-On Labs
Session A
MLOps Principles & Architecture
  • What is MLOps? Maturity levels (0 → 3) and organizational patterns
  • ML lifecycle: data → train → validate → deploy → monitor → retrain
  • MLOps vs DevOps: what's different about ML systems
  • Infrastructure overview: Docker, registries, orchestrators
  • Lab: Set up MLOps development environment (Docker, Git, MLflow)
Session B
Experiment Tracking & Model Versioning
  • MLflow: tracking experiments, parameters, metrics, and artifacts
  • MLflow Model Registry: staging, production, archiving
  • DVC (Data Version Control) for dataset versioning
  • Lab: Track a complete ML experiment lifecycle with MLflow
  • Lab: Version datasets and models with DVC + Git
02
Containerization & Model Serving
Live Coding + Hands-On Labs + Architecture Workshop
Session A
Docker for ML Engineers
  • Docker fundamentals: images, containers, Dockerfiles, volumes
  • Multi-stage builds for lean ML containers
  • Docker Compose for multi-service ML architectures
  • Lab: Containerize an ML model with all dependencies
  • Lab: Build a multi-container app (API + model + database)
Session B
Model Serving & APIs
  • FastAPI for ML inference endpoints
  • Batch vs real-time inference patterns
  • Model serialization: pickle, joblib, ONNX, TorchScript
  • Lab: Build a production inference API with FastAPI + Docker
  • Lab: Implement A/B testing with canary deployments

03
CI/CD for Machine Learning
Live Coding + Hands-On Labs + Pipeline Design
Session A
GitHub Actions for ML Pipelines
  • CI/CD principles applied to ML: test → validate → deploy
  • GitHub Actions: workflows, triggers, secrets, artifacts
  • Automated model testing: unit tests, integration tests, data validation
  • Lab: Build a GitHub Actions pipeline that trains, tests, and registers a model
  • Lab: Automated model quality gates (accuracy thresholds, drift checks)
Session B
Azure DevOps & ML Pipelines
  • Azure DevOps Pipelines for ML workloads
  • Azure ML Pipelines: components, datasets, compute targets
  • Multi-environment deployment: dev → staging → production
  • Lab: Build an Azure DevOps pipeline with model promotion
  • Lab: Implement automated retraining on data drift detection
04
Cloud MLOps — Azure, AWS & GCP
Live Demos + Hands-On Labs + Multi-Cloud Workshop
Session A
Azure ML & AWS SageMaker
  • Azure ML: workspaces, compute, endpoints, managed online inference
  • Azure ML Pipelines with components and designer
  • AWS SageMaker: training jobs, model registry, endpoints
  • Lab: Deploy a model to Azure ML managed endpoint
  • Lab: Deploy the same model to SageMaker endpoint
Session B
GCP Vertex AI & Multi-Cloud Patterns
  • GCP Vertex AI: pipelines, model registry, prediction
  • Comparing cloud platforms: cost, features, lock-in tradeoffs
  • Open-source MLOps stack: MLflow + Airflow + Kubernetes
  • Lab: Build a cloud-agnostic ML pipeline with MLflow
  • Lab: Kubernetes deployment with Helm charts for ML
05
Monitoring, Governance & Capstone
Capstone Project + Architecture Review + Presentations
Session A
Production Monitoring & Observability
  • Model monitoring: accuracy drift, data drift, concept drift
  • Evidently AI and Weights & Biases for monitoring
  • Infrastructure monitoring: latency, throughput, error rates
  • Lab: Build a monitoring dashboard with drift detection alerts
  • Automated retraining triggers and feedback loops
Session B
Capstone: End-to-End MLOps Pipeline
  • Team project: Design and implement a complete MLOps pipeline
  • Data ingestion → training → validation → deployment → monitoring
  • Architecture review: scalability, cost, security, compliance
  • Team presentations and expert feedback
  • Certification paths and next steps
